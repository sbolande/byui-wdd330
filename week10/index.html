Client-Side Form Validation

  • Reasons for validating forms:
    • We want to get the right data, in the right format
    • We want to protect our users' data
    • We want to protect ourselves
  • The two types of client-side Validation are built-in via HTML5 features, and JavaScript
  • HTML5 form controls:
    • required
    • minlength;maxlength
    • min;max
    • type
    • pattern
  • CSS helpers for validation:
    • :valid
    • :invalid
    • :out-of-range
